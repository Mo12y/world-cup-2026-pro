# 核心原则（22 场实盘血泪教训）

## 原则 1：锚点分散 — 铁律，不可违反

```
任何两个方案不得共享同一个致命锚点。
致命锚点 = 一个判断如果错误，会导致该方案直接死亡。

反例（7/5）：四个方案全部依赖"加拿大进球" → 加拿大 0 球 → 全灭
反例（7/6）：四个方案全部依赖"巴西不败" → 巴西 1-2 → 全灭
正例（7/7）：方案 A 押"强队赢" / 方案 B 押"都平" → A 命中
```

**执行**：设计完方案后，强制检查——如果框架的核心判断错了，有没有至少一个方案能活？

## 原则 2：方向 > 精确比分

22 场数据：
- 方向二串一期望命中率：~35-55%
- 精确比分二串一期望命中率：~2-7%
- 唯一连续盈利的类型：方向投注

**默认使用胜平负方向。** 仅当用户明确要求冲击高赔时才加入比分。小注博高赔可以，但不能作为主方案。

## 原则 3：EV 校验

每个方案计算：框架概率 × 赔率 vs 1.0。

```
正 EV：框架认为发生的概率 > 市场隐含概率 → 可以投
负 EV：框架认为发生的概率 < 市场隐含概率 → 避开或仅作保险
```

**不投负 EV 方案作为主方案。** 保险方案可以容忍轻微负 EV（保费性质）。

## 原则 4：互斥互补

方案 A 和方案 B 应覆盖不同的比赛场景。理想状态：

```
方案 A 覆盖场景 X（概率 ~40%）
方案 B 覆盖场景 Y（概率 ~30%）
方案 A + B 覆盖场景 X+Y（概率 ~70%）
方案 A 和 B 不可能同时死亡于同一个单场错误
```

## 原则 5：保险费原则

每轮至少有一个方案（占总预算 15-30%）押"框架可能错"的方向。这不是不相信框架——是 22 场数据告诉你框架会在 ~20% 的场次中方向错误。

# 执行步骤

**Step 1 — 读取框架分析结论**

从当日分析报告提取：
- 每场的 #1-#2 方向判断 + 概率
- 每场最具价值的单项（框架概率 > 市场隐含概率）
- 框架最脆弱的判断（"如果这个错了，一切都会错"）

**Step 2 — 计算 EV**

```
框架隐含公平赔率 = 1 / 框架概率
EV 比率 = 竞彩赔率 / 公平赔率
EV 比率 > 1.10 → 🔥 正 EV（被低估）
EV 比率 0.95-1.10 → ≈ 公平
EV 比率 < 0.95 → 🔴 负 EV（被高估）
```

**Step 3 — 设计互斥锚点**

```
确认方案 A 的核心锚点（如"两队强队都赢"）
确认方案 B 的核心锚点（如"至少一场平局"）
验证：A 和 B 的锚点互斥？✅ → 继续
验证：两个锚点之间有盲区？→ 标注盲区概率
```

**Step 4 — 分配金额**

按风格占比分配。主方案金额 = 预算 × 主方案占比，辅方案 = 预算 - 主方案金额。金额取偶数（2 元整数倍）。

**Step 5 — 输出 + 风险标注**
