# world-cup-2026-pro

2026 美加墨世界杯预测分析项目。经过 24 场比赛实盘验证、8 轮迭代（v1→v8），从一个 30 条规则的量化引擎退化成一个 3 条规则的决策过滤器——然后准确率反而提升了。

**核心发现：在淘汰赛中，量化分析产生的是负价值。讲的故事越多，预测越偏。**

> ⚠️ **研究项目**。不构成投注建议。竞彩有风险。

---

## 一句话总结

24 场比赛、30 条规则、8 轮迭代——最后的结论是：**别分析了。看 FIFA 排名，只在三个硬条件同时满足时才推翻它。**

## 框架进化故事（v1 → v8）

### 黄金期：v1-v3（前 7 场，100% 比分命中）

最初框架只有 ~10 条核心规则：攻击力预检、防守折扣、尊严进球审查、教练基线。简单，直接，有效。前 7 场 1/16 决赛的比分全部完美命中。

### 膨胀期：v4-v6.1（规则从 10 条涨到 30 条）

每一场预测错了，复盘就找到"为什么"，然后加一条规则堵漏洞。奥地利被零封→加防守折扣。加拿大三层次失败→加 AAS、CTI、双维度防线、大赛型门将、核心缺失折扣、NDA、USQ、HTA、单队 3+ 球、FRC。每条规则在复盘里都有完美的证据支持。合在一起，v6.1 在 1/8 决赛前两场 0/2 命中。

**v3(10条) 100% → v6.1(30条) 0%。** 相关系数 ≈ -0.85。

### 觉醒期：v7-v8（砍回 15 条 → 砍到 3 条）

v7 把框架分成预测层（15 条）和解释层（9 条），试图保留"有用的"规则。在 1/8 决赛中，v7 晋级方命中率 62.5%——比 FIFA 排名裸猜（75%）还差。

对抗式审查发现了真相：**v7 的分析在帮倒忙。** 两次方向错误（墨西哥海拔叙事、葡萄牙伊比利亚德比叙事）都是框架被一个"好故事"说服，主动推翻了正确的默认。

v8 废弃了 v1-v7 的所有量化规则。只留 3 条硬推翻条件。

## v8 决策过滤器

```
默认：FIFA 排名更高方晋级。

推翻默认 — 需 ≥2 条同时触发：
  □ ① 排名更高方有核心球员确定缺席
       → 核心 = 中场攻防枢纽 / 防线指挥官 / 进攻组织单一枢纽
  □ ② 排名较低方有 A 级防线
       → 淘汰赛 ≤1 场丢球 + 对 FIFA Top 15 零封记录
  □ ③ 排名较低方有顶级终结者（≥5球）+ 排名更高方淘汰赛连续丢球

不满足 → 维持默认。不分析。不叙事。不调整权重。
```

1/8 决赛三方对比：

| 系统 | 晋级方 | 规则数 | 核心 |
|------|:---:|:---:|------|
| v8 | **7/8 (87.5%)** | 3 | FIFA排名+硬推翻 |
| FIFA 排名 | 6/8 (75%) | 0 | 排名高者晋级 |
| v7 | 5/8 (62.5%) | 15 | 量化分析 |

v8 唯一错误：瑞士 pk 胜哥伦比亚（0-0，120分钟 + 点球）。任何简单系统都无法捕捉。

唯一正确推翻默认：巴西 vs 挪威。Paqueta 报销 + Haaland 5 球 → ≥2 条 → 预测挪威赢 → 命中。

## 8 轮迭代的 5 个核心教训

### 1. 规则越多，准确率越低

```
v1-v3: ~10条 → 7/7 (100%)
v4-v5: ~15条 → 2/9 (22%)
v6.1:   30条 → 0/2 (0%)
v7:     15条 → 5/8 (62.5%)
v8:      3条 → 7/8 (87.5%)
```

每条规则都在引入一个可以被现实推翻的假设。淘汰赛的高方差意味着更多假设 = 更多被打脸的机会。复盘写得再漂亮，加规则这件事本身就在伤害预测。

### 2. 叙事绑架——"好故事"是框架最大的敌人

| 场次 | 框架讲的故事 | 比赛怎么说 |
|------|------|------|
| 摩洛哥120分钟 | "体能差→加拿大有机会" | 摩洛哥下半场3球 |
| 墨西哥海拔 | "2200m→英格兰不行" | 英格兰36-38'双响 |
| 伊比利亚德比 | "5/5都预测进球→葡萄牙会进球" | 葡萄牙0球 |
| 比利时120分钟 | "4天恢复不够→低比分" | 比利时9分钟进球 |
| 挪威克星 | "4场样本不够→不算魔咒" | 挪威28年后再赢巴西 |

5 次叙事，5 次被打脸。4/5 次框架站在冷门方——因为冷门的"故事"比热门的"没有故事"更有分析价值。分析报告里冷门叙事占 80% 篇幅 → 篇幅转化为权重 → 权重推翻正确的默认。

### 3. 复盘和投注之间存在裂缝

7 份复盘中的核心投注教训：

| 复盘日期 | 核心结论 | 第二天投注 |
|------|------|------|
| 7/4 | "简单方向 > 精确比分" | 7/5: 全押精确比分二串一 |
| 7/5 | "锚点集中 = 全灭" | 7/6: 全押巴西不败 |
| 7/3 | "保守 Dutching 唯一盈利" | 再用也没再用了 |

复盘写得很漂亮，但每次设计投注方案时，复盘的教训被遗忘了。

### 4. 亏损主因不是预测不准，是亏损后加大赌注

```
7/2: +32 → 投入 90
7/3: +19 → 投入 60
7/5: -28 → 投入 28
7/6: -20 → 投入 20
7/7: +8  → 投入 12
7/8: -94 → 投入 94  ← 单日前一天的 8 倍
```

一周从 +51 到 -83。如果 7/8 只投 10-20 元（跟 7/7 一样），亏损可控。跳升到 94 元是毁灭性的。

### 5. 事后诸葛亮是优雅的毒药

复盘总是能找到"为什么错了"——然后用新规则堵漏洞。AAS 完美解释加拿大为什么被高估。FRC 完美解释摩洛哥为什么不累。HTA 完美解释加拿大为什么下半场崩盘。每条规则在当下都感觉完全正确。

问题是：这些规则是为了一场特定的比赛而创造的。它们在下一场比赛中要么不触发，要么触发但不改变预测，要么触发且让预测更差。**没有一条 v6.1 新增的规则在实际比赛中被独立验证过两次。**

v8 的反制措施：新规则需要反事实测试 + 3 场盲测期 + ≥3 次错误模式重复——三者全满足才能进入框架。这基本等于"再也不加规则"。
